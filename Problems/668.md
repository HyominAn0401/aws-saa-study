### 📝 668. 인스턴스 정기적 패치 
인스턴스 정기적 패치 및 보안 검색, 패치 상태에 대한 보고서 제공 필요 

---

### 🎯 1. 요구사항 분석
* **정기적 보안 검색**: -> Amazon Inspector 
* **정기적 EC2 인스턴스 패치**: -> AWS Systems Manager Patch Manager 
* **패치 상태 보고서**: -> Amazon Inspector

---

### 🔍 2. 핵심 서비스 분석 

| 서비스/용어명 | 핵심 특징 | 비고 |
| :--- | :--- | :--- |
| **Amazon Macie** | S3 버킷을 스캔해 민감한 데이터를 찾아내고 보호하는 서비스<br>머신러닝을 사용해 개인정보가 S3에 노출되어 있는지 자동으로 탐지 | 개인식별정보 탐지, 데이터 민감도 분석 | 
| **크론 작업** | 리눅스에서 특정 시간에 특정 작업을 자동으로 실행하게 만드는 스케줄러 | AWS에서 EventBridge가 서버리스로 수행 |
| **Amazon GuardDuty (위협탐지)** | AWS 계정 전체를 24시간 감지하는 지능형 CCTV | 악의적 활동 탐지, 계정 보안 위협 모니터링 |
| **AWS Systems Manager Session Manager** | SSH키 없이도 웹 브라우저에서 직접 EC2 인스턴스에 접속하게 해주는 서비스 | Bastion Host 없이 접속 |
| **Amazon Detective (보안 사고 조사)** | 보안 문제가 발생했을 때 원인 분석 | 보안 사고 근본 원인 분석 |
| **Amazon Inspector** | EC2나 컨테이너 이미지에 보안 취약점이 있는지 검사하는 서비스 | 취약점 스캔 |
| **AWS Systems Manager Patch Manager** | 수십 대의 EC2 인스턴스에 보안 패치나 업데이트를 자동으로 설치해주는 서비스 | |

---

### ✅ 3. 선택지 분석 및 풀이

- [ ] **A**: Amazon Macie + EC2 크론 작업 설정 
    - **이유**: Macie는 S3 버킷에 있는 데이터의 개인정보를 찾는 용도 
- [ ] **B**: GuardDuty + AWS Systems Manager Session Manager 
    - **이유**: GuardDuty는 계정 전체의 수상한 움직임을 찾는 용도로 EC2 인스턴스에서 취약점을 찾지 않음 
- [ ] **C**: Amazon Detective + EventBridge
    - **이유**: 보안 사고의 이유와 경로를 조사하는 용도 
- [X] **D**: Amazon Inspector + AWS Systems Manager Patch Manager
    - **결과**: **정답**
    - **이유**: 서버의 보안 취약점을 스캔하며 스캔 완료 후 인스턴스의 패치 상태를 리스트로 제공함
