### ✅ 3. 선택지 분석 및 풀이 

- [X] **A**: 읽기 전용 복제본 생성 
    - **결과**: **정답**
    - **이유**: 보고서 작업은 100% 읽기 작업으로 복제본을 만들면 운영 DB는 앱 트래픽을 처리하고 복제본은 보고서 쿼리만 처리하게 되어 부하가 격리됨 
- [ ] **B**: DynamoDB로 전환
    - **이유**: SQL 서버를 NoSQL로 바꾸는 건 **엄청난** 작업임. 
- [ ] **C**: 더 큰 인스턴스 선택(scale up)
    - **이유**: 성능은 좋아지지만 하나의 인스턴스 안에서 운영 트래픽과 보고서 쿼리가 자원을 나눠써야 해 근본적인 부하 분산이 이루어지지 않음 
- [ ] **D**: RDS를 Auto Scaling Group에 넣음
    - **이유**: 기술적으로 불가능함 -> 데이터 동기화 문제
