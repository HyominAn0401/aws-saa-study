### 📝 669. 워크로드의 예측 가능성
데이터를 DynamoDB에 저장하고 앱으로 데이터를 분석함. 데이터 워크로드는 일정하고 예측 가능하며 DynamoDB에 대해 예산 이하로 유지하길 원할 때 비용 효율적으로 충족하는 솔루션 찾기.

---

### 🎯 1. 요구사항 분석
* **데이터 워크로드가 일정하고 예측 가능**: 시간, 트래픽 규모, 처리 패턴이 미리 정해져 있어 거의 변하지 않음 

---

### 🔍 2. 핵심 서비스 분석

| 서비스/용어명 | 핵심 특징 | 비고 |
| :--- | :--- | :--- |
| **프로비저닝 모드** | 사용할 성능(RCU, WCU) 미리 예약해두는 방식 | 워크로드가 일정하고 예측 가능할 때 저렴 |
| **주문형 모드(On-Demand Mode)** | 성능을 미리 정하지 않고 요청이 들어온 만큼만 돈을 내는 방식<br>워크로드를 예측할 수 없거나 트래픽이 아주 가끔 발생하는 경우에 유리하나 요청당 단가는 프로비저닝 모드보다 비쌈 | 자동 무한 확장 |
| **DynamoDB Standard-IA** | 자주 접근하지 않는 데이터를 저장할 때 사용하는 저렴한 저장소 클래스 | 로그 데이터, 오래된 게시물 등 |
| **예상 워크로드 예약** | 선불로 결제하고 할인 받는 제도<br>최소 사용량이 확실할 때 비용을 매우 아낄 수 있음 | 프로비저닝 모드에서만 사용 가능 | 
| **RCU, WCU** | 프로비저닝 모드에서 성능을 측정하는 단위<br>RCU: 1초에 4KB 데이터를 1번 읽을 수 있는 능력<br>WCU: 1초에 1KB 데이터를 1번 쓸 수 있는 능력 | 성능 부족 -> 높이기<br>비용 투머치 -> 최적화 |

---

### ✅ 3. 선택지 분석 및 풀이

- [ ] **A**: 프로비저닝 모드 + DynamoDB Standard-IA + 예상 워크로드 예약 
    - **이유**: DynamoDB Standard-IA는 거의 접근하지 않는 데이터를 저장할 때 사용
- [X] **B**: 프로비저닝 모드 + RCU, WCU 지정
    - **결과**: **정답**
    - **이유**: 워크로드가 예측 가능할 때 주문형 모드보다 프로비저닝 모드를 사용해야 비용 절감 가능 + 미리 RCU WCU 지정
- [ ] **C**: 주문형 모드 + RCU, WCU 높게 설정
    - **이유**: 쓴 만큼 지불해야 하며 프로비저닝 모드보다 비용이 더 많이 듦 
- [ ] **D**: 주문형 모드 + 예약된 용량으로 WCU, RCU 지정 
    - **이유**: 쓴 만큼 지불해야 하며 프로비저닝 모드보다 비용이 더 많이 듦 

---

### 💡 핵심 요약
1. 예측 가능한 워크로드와 비용 절감 -> 프로비저닝 모드
2. 알 수 없는 워크로드, 서버리스 확장성 -> 주문형 모드(on-demand)
