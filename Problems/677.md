### 📝 677. 데이터 보호와 실수 방지 
중요한 데이터가 포함된 S3 버킷의 데이터가 실수로 삭제되지 않도록 보호해야 함. 이때 수행해야 하는 단계 조합은?

---

### 🎯 1. 요구사항 분석 (Requirements)
* **실수 방지 및 예방**: 권한이 있는 사용자라도 단순 클릭만으로 데이터를 영구 삭제할 수 없도록 추가 인증 단계 요구
* **데이터 복구력 확보**: 데이터가 삭제되거나 덮어쓰여져도 이전 상태의 데이터 보존해 즉시 롤백 가능해야 함
* **가용성 유지**: 보안이나 비용절감보다 데이터 존재 자체를 지키는 것이 최우선 

---

### 🔍 2. 핵심 서비스 분석 (AWS Documentation 기반)

| 서비스/용어명 | 핵심 특징 | 비고 |
| :--- | :--- | :--- |
| **S3 버전 관리** | 버킷에 업로드되는 객체에 고유 ID를 부여하고 히스토리 관리, 삭제 시 실제 데이터 대신 논리적 삭제 | 실수 복구를 위한 필수 기능
| **MFA 삭제** | 객체 영구 삭제나 버킷 버전 관리 상태 변경 시 MFA 인증 강제 | 물리적 이중 잠금으로 실수 차단 |
| **버킷 정책** | JSON 기반 접근 제어. 특정 액션(Delete)을 명시적으로 거부 가능 | 권한 관리 도구 |
| **기본 암호화** | 데이터 저장 시 서버 측에서 자동 암호화(SSE) 수행 | 데이터 유출 방지용 |
| **수명 주기 정책** | 시간 경과에 따라 객체 삭제나 스토리지 클래스 이동 자동화 | 비용 최적화 도구 |

### ✅ 3. 선택지 분석 및 풀이 (Analysis)

- [X] **A**: 버전 관리 활성화
    - **결과**: **정답**
    - **이유**: 사용자가 삭제해도 이전 버전이 남아있어 롤백 가능
- [X] **B**: MFA 삭제 
    - **결과**: **정답**
    - **이유**: 객체 영구 삭제 시 반드시 MFA 기기 인증 강제   
- [ ] **C**: 버킷 정책 생성 
    - **이유**: 삭제 권한 제한할 수 있지만 권한이 있는 관리자가 삭제할 수 있음 
- [ ] **D**: 기본 암호화
    - **이유**: 저장된 데이터를 암호화하는 용도
- [ ] **E**: 수명 주기 정책
    - **이유**: 일정 기간 후 데이터를 삭제하거나 이동시키는 기능

### 💡 핵심 요약
1. S3 삭제 방지: 버전 관리(복구용) + MFA 삭제 (예방용)
2. MFA 삭제는 MFA 기능을 지운다는 뜻이 아니라 삭제할 때 MFA가 필요하다는 뜻 (오답의 이유)
3. 암호화와 수명 주기는 오답 후보임
